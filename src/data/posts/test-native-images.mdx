# Testing Native Astro Image Components

This is a test post to verify how Astro 5 handles images natively in MDX files, replacing Hugo's `imgs` shortcode functionality.

## Single Responsive Image

<Image
  src={logoSymbol}
  alt="CTO4.AI logo symbol"
  widths={[200, 400, 800]}
  sizes="(max-width: 768px) 200px, (max-width: 1200px) 400px, 800px"
  class="mx-auto"
/>

## Picture Component with Multiple Formats

<Picture
  src={logoSymbol}
  alt="CTO4.AI logo symbol with multiple formats"
  widths={[200, 400, 800]}
  formats={['avif', 'webp', 'png']}
  sizes="(max-width: 768px) 200px, (max-width: 1200px) 400px, 800px"
  class="mx-auto my-8"
/>

This replaces the Hugo shortcode pattern:

```
{{< imgs kind="page" size="xl" imgs="logo-symbol.png" >}}
```

With native Astro components that provide:

- Automatic format optimization (WebP, AVIF)
- Responsive sizing with `widths` and `sizes`
- Layout shift prevention
- Better performance and smaller bundle sizes
